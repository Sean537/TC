# ANSI 转义码完整参考

本文档提供了 TC.hpp 库中使用的 ANSI 转义码的详细参考，包括颜色、样式、光标控制等。

## 基本格式

ANSI 转义序列的基本格式为：`\033[<参数>m`，其中 `\033` 是 ASCII 转义字符（ESC），`[` 是控制序列引导符，`<参数>` 是控制参数，`m` 是序列终止符。

## 文本样式控制

| 代码 | 效果 | 兼容性 |
|------|------|--------|
| `\033[0m` | 重置所有样式 | 所有终端 |
| `\033[1m` | 粗体/加粗 | 大多数终端 |
| `\033[2m` | 微弱/淡色 | 部分终端 |
| `\033[3m` | 斜体 | 部分终端 |
| `\033[4m` | 下划线 | 大多数终端 |
| `\033[5m` | 慢速闪烁 | 部分终端 |
| `\033[6m` | 快速闪烁 | 很少终端 |
| `\033[7m` | 反色（前景色与背景色交换） | 大多数终端 |
| `\033[8m` | 隐藏（不可见文本） | 部分终端 |
| `\033[9m` | 删除线 | 部分终端 |
| `\033[21m` | 双下划线/关闭粗体 | 部分终端 |
| `\033[22m` | 关闭粗体和微弱 | 大多数终端 |
| `\033[23m` | 关闭斜体 | 部分终端 |
| `\033[24m` | 关闭下划线 | 大多数终端 |
| `\033[25m` | 关闭闪烁 | 部分终端 |
| `\033[27m` | 关闭反色 | 大多数终端 |
| `\033[28m` | 关闭隐藏 | 部分终端 |
| `\033[29m` | 关闭删除线 | 部分终端 |

## 256色模式

ANSI 支持 256 色模式，格式为：

- 前景色：`\033[38;5;<颜色代码>m`
- 背景色：`\033[48;5;<颜色代码>m`

其中 `<颜色代码>` 范围为 0-255：
- 0-7: 标准颜色
- 8-15: 亮色
- 16-231: 6×6×6 RGB 颜色立方体
- 232-255: 灰度色阶

## RGB真彩色

ANSI 支持 24 位真彩色（1670 万色），格式为：

- 前景色：`\033[38;2;<r>;<g>;<b>m`
- 背景色：`\033[48;2;<r>;<g>;<b>m`

其中 `<r>`, `<g>`, `<b>` 分别是 RGB 三个分量的值（0-255）。

## 光标控制

| 代码 | 功能 |
|------|------|
| `\033[H` | 将光标移动到屏幕左上角（1,1） |
| `\033[<行>;<列>H` | 将光标移动到指定位置 |
| `\033[<n>A` | 光标上移 n 行 |
| `\033[<n>B` | 光标下移 n 行 |
| `\033[<n>C` | 光标右移 n 列 |
| `\033[<n>D` | 光标左移 n 列 |
| `\033[<n>E` | 光标下移 n 行，并移到行首 |
| `\033[<n>F` | 光标上移 n 行，并移到行首 |
| `\033[<n>G` | 光标移动到当前行的第 n 列 |
| `\033[s` | 保存光标位置 |
| `\033[u` | 恢复保存的光标位置 |

## 清除屏幕

| 代码 | 功能 |
|------|------|
| `\033[J` | 清除从光标到屏幕末尾的内容 |
| `\033[1J` | 清除从光标到屏幕开头的内容 |
| `\033[2J` | 清除整个屏幕 |
| `\033[3J` | 清除整个屏幕和滚动缓冲区（部分终端支持） |
| `\033[K` | 清除从光标到行末的内容 |
| `\033[1K` | 清除从光标到行首的内容 |
| `\033[2K` | 清除整行 |

## 特殊控制

| 代码 | 功能 |
|------|------|
| `\033[?25l` | 隐藏光标 |
| `\033[?25h` | 显示光标 |
| `\033[?47h` | 切换到备用屏幕缓冲区 |
| `\033[?47l` | 切换回主屏幕缓冲区 |
| `\033[?1049h` | 切换到备用屏幕缓冲区并保存当前屏幕 |
| `\033[?1049l` | 恢复保存的屏幕并切换回主屏幕缓冲区 |

## 组合使用

ANSI 转义码可以组合使用，多个参数用分号分隔：

```
\033[1;31;43m  // 粗体、红色文本、黄色背景
```

## 兼容性注意事项

1. **Windows 支持**：
   - Windows 10 以前的 CMD 不原生支持 ANSI 转义序列
   - Windows 10 及以上版本的控制台支持大部分 ANSI 转义序列
   - TC.hpp 在 Windows 平台上**不使用** ANSI 转义序列，而是直接使用 Windows Console API (Win32 API) 实现所有终端控制功能

2. **终端类型**：
   - 不同的终端模拟器对 ANSI 转义序列的支持程度不同
   - 现代终端（如 iTerm2、GNOME Terminal、Windows Terminal）支持大多数功能
   - 旧终端或简单终端可能只支持基本功能

3. **颜色深度**：
   - 并非所有终端都支持 256 色或真彩色
   - 不支持的终端会降级到最接近的颜色

## 在 TC.hpp 中的使用

TC.hpp 库封装了这些 ANSI 转义码（在非Windows平台上）和Windows Console API（在Windows平台上），提供了统一的跨平台接口：

```cpp
// 使用预定义宏
tc::println(TCOLOR_RED, "红色文本", TCOLOR_RESET);

// 使用 RGB 颜色
tc::println(TCOLOR_RGB(255, 128, 0), "橙色文本", TCOLOR_RESET);

// 使用光标控制
tc::printer().moveCursor(10, 5).print("在位置(10,5)打印");
```

TC.hpp 会自动处理不同平台的兼容性问题，使开发者可以专注于功能实现而不必担心底层细节。在Windows平台上，所有颜色、字体样式和终端控制功能都通过Win32 API直接实现，因此能够完全支持，不受终端对ANSI转义序列支持程度的限制。
